# UX / IA（画面設計・文言設計）

## 1. 体験のゴール
- 1日合計5分未満で「学び→実践→整え」が回る
- 連続より復帰を称える
- “儀礼感”は出すが、重くしない

## 2. IA（画面一覧）
MVP（タブ + 追加画面）
- Home（今日の進捗）
- Check-in（朝）
- Learn（今日の学び）
- Night（整え）
- Log（週次）
- Settings（通知・課金・トーン）

## 3. ユーザーフロー
### 初回（オンボーディング3枚）
1) このアプリは「日常実践の補助輪」  
2) 三密（身・口・意）の説明（短い）  
3) 途切れてOK（復帰が偉い）

### 毎日（最短）
Home → 朝（3分） → 昼（30秒） → 夜（45秒）

## 4. 画面仕様（MVP）
### Home
- 今日：朝/昼/夜の状態（未/済）
- 今日の学びタイトル（タップでLearnへ）
- 今週の三密バランス（身/口/意の回数）
- CTA
  - 「朝を整える」
  - 「学びを見る」
  - 「夜を閉じる」

### Morning Check-in（朝）
3カード（身・口・意）を縦に並べる
- 各カード：
  - タイトル（身/口/意）
  - ガイド（10〜60秒）
  - 「完了」ボタン
- 最後に「今日の行い」を1つ選ぶ（身/口/意から）

### Learn（学びカード）
- 今日の言葉（用語）
- 30秒説明
- たとえ（生活）
- 今日の行い（候補）
- 任意：ミニ問い（1問）
- ボタン：保存（ブックマーク）

### Night Close（夜：整え）
3ステップを1画面で縦に
1) 懺悔：今日いちばん乱れたのは？（身/口/意）
2) 発願：明日1つだけ整えるなら？（身/口/意）
3) 回向：誰に回す？（自分/家族/チーム/みんな）
- 任意：1行メモ
- 完了CTA：「閉じる」

### Log（週次）
- カレンダー or 週次カード
- 週の三密回数（身/口/意）
- 乱れ（懺悔）の傾向（身/口/意）
- 発願の傾向（身/口/意）
- ログ一覧を作る場合は FlatList で仮想化する
※診断っぽく言わない。「傾向」として静かに出す

### Settings
- 朝通知時刻
- 夜通知時刻
- 通知ON/OFF
- トーン（neutral/gentle）
- プレミアム（課金導線）

## 5. 通知（コピー案）
朝：
- 「3分だけ整えよう（身・口・意）」
夜：
- 「45秒で閉じよう（懺悔→発願→回向）」
復帰（途切れ翌日）：
- 「戻れたら十分。今日は短くでもOK」

## 6. トーンガイド（UX）
- 断定しない（効果保証しない）
- 責めない（できない日を前提にする）
- 短い（ワンセンテンス）
- 敬意（軽薄なノリは避ける）
