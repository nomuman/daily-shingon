# アプリ企画・基本設計（真言密教：学び×実践サポート）

---

## 0. この資料の目的
- 真言密教（真言宗）を「理解し、日々の生活で実践し続ける」ためのアプリを作る。
- 宗教の代替ではなく、**日常実践のサポート**に徹する。
- 実装前に、コンセプト / Who-What-How / 基本設計（情報設計・データ設計・課金設計）を固める。

---

## 1. コンセプト（1行）
**三密（身・口・意）のチェックインを毎日回し、学びを“今日の一つの行い”に落として積み上げるアプリ。**

### 背景（真言密教の観点）
- 真言系の説明では、三密は「身＝印（身の働き）」「口＝真言（言葉の働き）」「意＝観想（心の働き）」を重ねていく実践として語られる。
- 高野山の英語パンフでも、三密は「身体（mudrā）・言葉（mantra）・心（meditation）」を一緒に行う実践として説明される。
- 阿字観は真言系で重要な瞑想法として紹介され、一般にも体験可能・取り組みやすいと説明されることがある。
- 日常五心（仁和寺の解説）には「反省の心（すみません）」が含まれ、三密/三業（身口意の乱れ）との関連が語られている。

> 注：ここで扱うのは「生活実践の入口」。灌頂や口伝などの専門領域はアプリの範囲外（非代替）。

---

## 2. Who / What / How

### Who（誰のため）
- 真言密教に興味があり、空海（弘法大師）の教えを学びたい人
- でも、読書や動画だけだと散らばって理解できない / 続かない人
- 寺院体験（高野山など）にも関心があるが、日常での型がない人

### What（何を提供する）
1. **毎日3分の実践（朝）**：身・口・意を整える最小ルーティン
2. **毎日1枚の学び（昼）**：用語・思想を短く理解し、今日の行いに変換
3. **45秒の整え（夜）**：懺悔 → 発願 → 回向 で1日を閉じる
4. **任意の深掘り**：阿字観タイマー（3/5/10分）・コース学習（課金）

### How（どうやって実現する）
- “正しさの不安”を減らすために、テキストは短く・出典方針を明確化し、将来的に寺院/僧侶監修の導線を用意する。
- 継続の中心は「短い」「迷わない」「途切れても戻れる」。
- 実践の成果を断言しない（スピリチュアル断定は避ける）。代わりに、**実践の積み上げ**と**自分の変化の気づき**を可視化する。

---

## 3. 体験（ユーザーが毎日やること）

### 朝（3分）：三密チェックイン
- 身（10〜30秒）：姿勢を整える / 合掌・一礼（チェック）
- 口（10〜30秒）：短い真言 or 今日の誓い（チェック）
- 意（60秒）：1分観想（ガイド） or 呼吸に戻る（チェック）
- 最後に「今日の行い」1つ選ぶ（例：返信前に一呼吸）

### 昼（30秒）：学びカード
- 用語・思想（短文）→ たとえ → 今日の行い → ミニ問い1つ（任意）
- 例）三密 / 三業 / 加持（かじ） / 阿字観 / 曼荼羅 / 真言…など

### 夜（45秒）：整え（懺悔→発願→回向）
- 懺悔：今日いちばん乱れたのは？（身/口/意 3択）
- 発願：明日1つだけ整えるなら？（身/口/意 3択）
- 回向：功徳を誰に回す？（自分/家族/チーム/みんな）

> 日記は不要。1行メモは任意（重くしない）。

---

## 4. プロダクト原則（設計の掟）
1. **短い**：朝3分・夜45秒で完了
2. **型がある**：迷わない（毎日ほぼ同じ流れ）
3. **続けやすい**：連続より復帰を称える（途切れOK）
4. **断言しない**：効果保証しない。実践の支援に徹する
5. **敬意と正確さ**：出典方針を明記し、監修の余地を残す
6. **プライバシー最優先**：個人ログは端末優先、同期はオプション

---

## 5. 情報設計（IA）・画面構成（MVP）

### 5.1 画面一覧（最小 5画面）
1) Home（今日の進捗）
- 朝：未/済
- 昼：学びカード（未/済）
- 夜：未/済
- 今週の三密バランス（身/口/意）

2) Morning Check-in（朝）
- 身/口/意 の3カード（各「ガイド」or「完了」）
- 今日の行い（1つ選ぶ）

3) Learn（学び）
- 今日のカード（短文）
- 任意：ミニ問い1つ
- 既読・保存（ブックマーク）

4) Night Close（夜）
- 懺悔（身/口/意）
- 発願（身/口/意）
- 回向（対象選択）
- 任意：1行メモ

5) Log / Insights（ログ）
- カレンダー or リスト
- 身/口/意 の達成回数
- 「乱れやすい傾向（口が多い等）」を軽く提示

### 5.2 オンボーディング（3枚）
- ① このアプリは何か：学びと実践の“入口”
- ② 三密の説明：身/口/意を整える短い型
- ③ 途切れてOK：戻ることが実践（連続より復帰）

## 6. 機能要件（MVP → v1）

### MVP（まず出す）
- 朝チェックイン（身/口/意 + 今日の行い）
- 学びカード（1日1枚）
- 夜クローズ（懺悔→発願→回向）
- ログ（簡易）
- 通知（朝/夜）

### v1（伸ばす）
- 阿字観タイマー（3/5/10分、初心者ガイド）
- 学びコース（30日/90日）
- ブックマーク / 検索
- 週間レビュー（1分）
- 有料：音声ガイド / 深掘り教材 / コース解放

### 将来（差別化の本丸）
- 寺院/僧侶監修コンテンツ（信頼性）
- 体験導線（阿字観体験、法話、参拝プラン）
- コミュニティ（ただし宗教トラブル回避のモデレーション必須）

---

## 7. 非機能要件
- オフライン動作（最低限）
- 端末内保存を基本に（個人ログの外部送信を最小化）
- 表示は「静か・簡素・儀礼感」（派手なゲーミフィケーションは控えめ）
- アクセシビリティ：文字サイズ、ダークモード

---

## 8. データ設計（最小）

### 8.1 エンティティ
#### Entry（朝/夜の実践）
- id: string
- date: 'YYYY-MM-DD'
- slot: 'morning' | 'night'
- bodyDone: boolean
- speechDone: boolean
- mindDone: boolean
- actionPick?: 'body'|'speech'|'mind' (今日の行い)
- sange?: 'body'|'speech'|'mind' (懺悔)
- hatsugan?: 'body'|'speech'|'mind' (発願)
- eko?: 'self'|'family'|'team'|'all' (回向)
- note?: string
- createdAt, updatedAt

#### LearnCard
- id: string (day or slug)
- title: string
- body: string (短文)
- practiceOptions: string[] (今日の行い候補)
- quiz?: { question: string, choices: string[], answerIndex: number }
- tags: string[]

#### UserSettings
- reminderMorningTime: 'HH:mm'
- reminderNightTime: 'HH:mm'
- tone: 'neutral'|'gentle'
- premium: boolean

### 8.2 保存方針
- MVPは端末内（AsyncStorage / SQLite）でOK
- 同期はv2以降（ログは個人情報になりやすいので慎重に）

---

## 9. 通知設計（継続のための最小）
- 朝：指定時刻に「3分で整える（身・口・意）」
- 夜：指定時刻に「45秒で閉じる（懺悔→発願→回向）」
- 途切れた翌日：責めない文言で「戻ろう」（連続を煽らない）

---

## 10. 学びカリキュラム（たたき台）
### 30日（入口）
- Week1：三密（身口意）と、日常への落とし方
- Week2：三業（身口意の乱れ）→整える方法
- Week3：真言・印・観想・曼荼羅の「役割」理解（深くやりすぎない）
- Week4：阿字観の導入、生活との接続（忙しい人向けに短く）

> 学びは必ず「今日の行い（1つ）」に変換して終える。

---

## 11. マネタイズ（継続と両立）
### 無料
- 三密チェックイン（朝/夜）
- 学びカード（基本）
- 簡易ログ

### 有料（サブスク/月・年）
- 90日コース（体系化）
- 阿字観の音声ガイド（鐘/読導）
- 週間レビュー（洞察）
- 学びカードのアーカイブ・検索・タグ強化
- 将来：監修コンテンツ（寺院協力）

### 課金導線（UX）
- “無料で習慣が回る”を先に作る（D7/D30優先）
- 7日連続や「復帰3回」など、自己効力感が出たタイミングでプレミアム提示

---

## 12. リスクと対策（重要）
1) 宗教的正確性・誤解
- 対策：出典方針・監修方針を明記。断定や煽りを避ける。専門領域はアプリ外へ誘導。

2) 効果保証に見える表現
- 対策：「必ず良くなる」等は避け、実践の支援と気づきの記録に徹する。

3) ユーザーログのセンシティブ性
- 対策：端末保存優先。クラウド同期はオプション。削除機能を明確化。

---

## 13. KPI（最初に見る指標）
- D1 / D7 / D30 継続
- 朝チェックイン完了率、夜クローズ完了率
- 途切れ後の復帰率（最重要）
- 学びカード完了率
- 無料→有料転換率、解約率
